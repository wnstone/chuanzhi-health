# 这个项目是什么

黑马程序员「传智健康」教学项目(体检预约管理系统),老式 SSM + Dubbo 分布式架构(2021 年课件)。
同学只需要跑 **chapter10**(最后一章,包含前面全部功能)。

技术栈:Spring 5.0.5 + SpringMVC + MyBatis 3.4.5 + Dubbo 2.6.0 + ZooKeeper + Redis + MySQL。

# 一句话诊断

> ✅ **已验证(2026-07-09)**:用 JDK 8 + Maven 在本机把 chapter10 六个模块全部 `BUILD SUCCESS` 编译通过——**代码本身零问题**,同学那边的「一堆 bug」就是环境没配对。

同学发来的 `源代码.zip` 是**干净的课件原版**(全部文件时间戳 2021-2023,没有任何 2026 年改动)——他用 Trae 改坏的代码没打包进来。
所以「一堆 bug」大概率不是代码问题,而是**环境没配对**。已知的三个环境杀手:

| 杀手       | 表现                                          | 解法            |
| -------- | ------------------------------------------- | ------------- |
| JDK 版本太新 | 项目要 **JDK 8**,装了 JDK 17/21/23 直接编译/启动失败     | 用便携 JDK 8(见下) |
| 中文路径     | JDK 8 老启动器放中文路径下报 `could not find java.dll` | 工具链放纯英文路径     |
| 中间件没起    | ZooKeeper / Redis / MySQL 没启动,启动项目就连不上报错    | 按下面顺序起        |

另有一个课件自带的小 bug 已修:`health_service_provider/src/main/resources/spring-dao.xml` 里数据库 URL 被换行断成三行(chapter09 原版同款毛病),已合并回一行。

# 给同学机器排错的检查单

1. JDK 是不是 8?`java -version` 看开头是不是 `1.8`。不是 = 装 JDK 8 或用便携版。
2. 项目路径有没有中文?有 = 挪到纯英文路径(如 `D:\health\`)。
3. ZooKeeper、Redis、MySQL 三个都起了吗?少一个都会报连接错。
4. provider 起了吗?后台报 `No provider available` = provider 没起或 ZooKeeper 没通。
5. 数据库密码对不对?`spring-dao.xml` 里写的是 root/root,和本机 MySQL 不一致就改配置。
6. 让 Trae 改过的代码**全部还原**(重新解压课件原版最干净)——报错根源在环境,改代码只会越改越乱。
